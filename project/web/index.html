Infraestructura

Internet: La Red Global

Internet es una red global descentralizada de redes interconectadas que utilizan el protocolo TCP/IP para comunicarse. Es la infraestructura técnica que permite el intercambio universal de información, facilitando servicios como la World Wide Web y la transferencia de datos en tiempo real.

Evolución Tecnológica

Un poco de Historia

1961

Leonard Kleinrock publica la primera teoría sobre conmutación de paquetes, el fundamento de la transmisión de datos moderna.

Se envía el primer mensaje entre computadoras de UCLA y Stanford, marcando el nacimiento práctico de la red ARPANET.

1969
1983

Adopción oficial del protocolo TCP/IP por ARPANET. Este suceso marca el inicio técnico de la red global interconectada.

Tim Berners-Lee publica su propuesta para un sistema de gestión de información descentralizado, naciendo la World Wide Web.

1989
2006

Explosión de las redes sociales y expansión global de los smartphones, transformando Internet en una plataforma ubicua y social.

Protocolos de Comunicación

Es un sistema de reglas que permite que dos o más entidades de un sistema de comunicaciones transmitan información a través de cualquier tipo de variación de una cantidad física. Estas son las reglas o estándares que definen la sintaxis, la semántica y la sincronización de la comunicación y los posibles métodos de recuperación de errores.

Puertos de Red

En la suite de protocolos de Internet, un puerto es un punto final (endpoint) de comunicación en un sistema operativo. Mientras que el término también se usa para conectores físicos, en software es un constructo lógico que identifica un proceso específico o tipo de servicio de red.

Un puerto siempre está asociado con una dirección IP y un tipo de protocolo, completando la dirección de origen o destino de una sesión de comunicación. Se identifica mediante un número de 16 bits.

Cliente - Servidor

En una arquitectura Cliente-Servidor existe un servidor y múltiples clientes que se conectan al servidor para recuperar todos los recursos necesarios para funcionar, en este sentido, el cliente solo es una capa para representar los datos y se detonan acciones para modificar el estado del servidor, mientras que el servidor es el que hace todo el trabajo pesado.
